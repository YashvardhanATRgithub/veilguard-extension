Configure & Auto-start Ollama

VeilGuard needs Ollama to allow browser extension requests. Run these once — Ollama will auto-start on every boot with the right settings.

If installed via Homebrew

1. Add the environment variable to your shell (one-time):

echo 'export OLLAMA_ORIGINS="*"' >> ~/.zshrc && source ~/.zshrc

2. Enable auto-start as a background service:

brew services start ollama

Ollama will now start automatically on every login. No terminal needed.

If installed via .dmg (desktop app)

1. Set the env var for GUI apps (one-time):

launchctl setenv OLLAMA_ORIGINS "*"

2. Open Ollama app → click menu bar icon → Preferences → enable "Launch at login".

3. Restart the Ollama app to apply the env var.

Manual / one-time start

If you just want to start it now without auto-launch:

OLLAMA_ORIGINS='*' ollama serve
✅ Ollama should now be running. Verify: curl http://127.0.0.1:11434/

Configure & Auto-start Ollama

VeilGuard needs Ollama to allow browser extension requests. Set this up once and Ollama will auto-start with the right settings.

Permanent setup

Add system environment variable

1. Open Start menu → search "Environment Variables" → click "Edit the system environment variables"

2. Click "Environment Variables" button

3. Under System variables, click New:

Variable name: OLLAMA_ORIGINS
Variable value: *

4. Click OK to save. Restart the Ollama system tray app.

Ollama auto-starts with Windows by default via the system tray. This env var ensures VeilGuard can communicate with it.

Quick start (one-time, not persistent)

Or start from PowerShell for this session only:

$env:OLLAMA_ORIGINS='*'; ollama serve

Configure & Auto-start Ollama

Permanent setup with systemd

1. Add the CORS override to the Ollama service:

sudo systemctl edit ollama

2. In the editor, add these lines and save:

[Service]
Environment="OLLAMA_ORIGINS=*"

3. Restart and enable auto-start:

sudo systemctl restart ollama && sudo systemctl enable ollama

Ollama will now auto-start on every boot with the right permissions.

Troubleshooting

Ollama is installed but shows "Offline"

Ollama needs to be running as a background service. Just having it installed is not enough. Start it using the instructions in Step 3 above for your OS.

Model pull is very slow

The qwen2.5:1.5b model is about 1GB. On a slow connection this may take a few minutes. You can track progress in your terminal.

Extension still shows "Offline" after starting Ollama

Wait ~10 seconds or click Re-check. If it still shows offline, ensure Ollama is running on the default port 11434. Check by running: curl http://127.0.0.1:11434/ — it should respond with "Ollama is running".

"Model load failed" or 403 errors

This means Ollama is blocking requests from the browser extension. You must start Ollama with OLLAMA_ORIGINS='*' to allow cross-origin requests. See Step 3 in your OS tab above.

Can I use a different model?

Yes! Pull any model with ollama pull model-name and select it from the dropdown in the VeilGuard popup. Larger models are more accurate but use more RAM and are slower.
